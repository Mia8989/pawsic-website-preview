# Wound Basics 101 — Make.com Automation Spec

**Purpose:** When a clinician registers via the Jotform on https://pawsic.org/wound-basics-101/, automatically create their free MemberPress account on memberships.pawsic.org and send the welcome email with watch-page link.

## 2. Trigger: Jotform Webhook

**Option A (recommended) — Configure inside Jotform:**
1. In Jotform → My Forms → Wound Basics 101 → Settings → Integrations → Webhooks
2. Paste your Make.com webhook URL
3. Jotform sends a POST request on every submission

**Option B — Make's native "Jotform > Watch New Submissions" trigger:**
- Connects directly via OAuth, no manual webhook setup needed
- Slightly delayed (polls every ~15 min on free Make plan)

## 3. Make Scenario Modules

### Module 1: Trigger
- **Jotform → Watch Form Submissions** (or **Webhooks → Custom Webhook**)

### Module 2: Router / Filter — Verify privacy agreement
- Continue only if `q11_iAgree` = "I agree"

### Module 3: HTTP → MemberPress REST API — Create Member
- **Method:** `POST`
- **URL:** `https://memberships.pawsic.org/wp-json/mp/v1/members`
- **Headers:**
  ```
  MEMBERPRESS-API-KEY: <your-memberpress-api-key>
  Content-Type: application/json
  ```
- **Body (JSON):**
  ```json
  {
    "email": "{{3.q3_email}}",
    "username": "{{3.q3_email}}",
    "password": "PAWSIC2026",
    "first_name": "{{3.q1_firstName}}",
    "last_name": "{{3.q2_lastName}}",
    "send_welcome_email": false,
    "memberships": [1],
    "profile": {
      "mepr_phone": "{{3.q4_phone}}",
      "mepr_job_title": "{{3.q5_jobTitle}}",
      "mepr_organization": "{{3.q6_organization}}",
      "mepr_country": "{{3.q7_country}}",
      "mepr_state": "{{3.q8_stateProvince}}",
      "mepr_referral_source": "{{3.q9_howDidYou}}"
    }
  }
  ```

> **Confirm the free membership ID before going live.** The free plan ID on memberships.pawsic.org is most likely `1` (per PAWSIC infrastructure docs the premium IDs are 931, 4303, 4212, 4125, 498, 3781, 1073 — so free is a different ID). Pull the list of plans with `GET /wp-json/mp/v1/memberships` and confirm.

> **Username and temp password:** username is set to the registrant's email; temp password is `PAWSIC2026`. These are displayed in the autoresponder email (`wound-basics-101-autoresponder.html`) so the registrant can log in immediately. The email prompts them to change the password after first login. If you change the temp password here, update it in the autoresponder HTML as well — they must stay in sync.

> **Username collision:** if you use email as username and the email already exists in MemberPress, the API will return an error. Handle this in Module 4 (Router) by checking for the existing-user error and skipping account creation; still send the watch-link email and use their existing password (point them to "forgot password" if needed).

### Module 4: Custom WordPress endpoint — Send password reset link
Two options:

**Option A:** Use the standard WordPress lost-password flow:
- `POST https://memberships.pawsic.org/wp-login.php?action=lostpassword`
- Form-encoded body: `user_login={{3.q3_email}}`
- This triggers WordPress's stock password-reset email

**Option B (cleaner):** Use a custom WPCode snippet that generates a one-time login link. Recommended if you want to send a branded "Welcome + Watch Now" email. Reuse the same pattern as the existing `wpcode-course-reminders.php` snippet.

### Module 5: Send branded welcome email (recommended)
Use the `pawsic-memberpress-email` skill template. Required content:

- **From:** `PAWSIC <info@pawsic.org>`
- **Subject:** `You're in — watch Wound Basics 101 now`
- **Body must include:**
  - Personal greeting (First Name)
  - Direct watch link: `https://pawsic.org/wound-basics-101/watch/`
  - Their MemberPress login: `https://memberships.pawsic.org/login/`
  - "Set your password" reset link
  - 72-hour window reminder (June 22 to June 24, 2026)
  - Certificate-of-Completion reminder
  - Plain-text unsubscribe + Privacy Policy footer

### Module 6 (optional): Mailchimp — Add to newsletter audience
- Only fire if `q10_newsletter` = "Yes"
- Mailchimp module: **Add a Subscriber to a List**
- List: PAWSIC Newsletter audience
- Tags: `wound-basics-101`, `free-member`, source tag from `q9_howDidYou`

### Module 7: Google Sheets — Log registration
- Append row to a tracking sheet for the registration count + source attribution
- Columns: timestamp, email, name, organization, role, country, source, opt-in

### Module 8 (optional): Slack notification
- Channel: PAWSIC internal ops
- Message: `New Wound Basics 101 registration: {{name}} ({{org}}) — Total today: {{count}}`

## 4. Error Handling

| Failure | Behavior |
|---------|----------|
| Email already exists in MemberPress | Skip member creation, still send watch-link email and add tag |
| MemberPress API timeout | Retry 3x with 30s backoff, then send fallback email with watch link + "we'll set up your account shortly" |
| Mailchimp 4xx | Log to error sheet, do NOT block welcome email |
| Privacy checkbox missing | Reject (Make Filter blocks before Module 3) |

Set up a Make scenario-level error handler that logs failed runs to a separate sheet and pings Slack on consecutive failures.
